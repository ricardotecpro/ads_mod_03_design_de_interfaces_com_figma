# Quiz 12: Variantes e Propriedades

1. O que são "Variantes" no Figma?
    - [ ] Vírus de computador.
    - [x] Uma forma de agrupar componentes similares (como diferentes estados de um botão) em um único conjunto organizado.
    - [ ] Diferentes tipos de fontes.

2. Como se chama o contêiner pontilhado que agrupa várias variantes?
    - [ ] Group Set.
    - [x] Component Set.
    - [ ] Frame Box.

3. As "Properties" (Propriedades) de um componente permitem:
    - [ ] Aumentar a velocidade do mouse.
    - [x] Alternar entre diferentes visuais ou estados do componente através de menus no painel lateral.
    - [ ] Mudar a senha do arquivo.

4. Uma "Boolean Property" (Propriedade Booleana) cria no painel lateral:
    - [ ] Uma caixa de texto.
    - [x] Uma chave de liga/desliga (Toggle) para mostrar ou esconder elementos.
    - [ ] Uma lista de 50 itens.

5. A propriedade do tipo "Instance Swap" serve para:
    - [x] Substituir rapidamente uma instância interna (como um ícone) por outra da biblioteca.
    - [ ] Trocar as cores do frame.
    - [ ] Mudar o nome do arquivo.

6. Qual o benefício de usar Variantes em vez de componentes soltos no painel Assets?
    - [ ] Ganhar mais espaço em disco.
    - [x] Organizar melhor a biblioteca, agrupando elementos relacionados sob um único nome.
    - [ ] O Figma roda mais rápido.

7. No nome de uma camada para criar variantes automaticamente, usamos qual padrão (ex: Status=Default, Type=Primary)?
    - [x] Nome da Propriedade = Valor da Propriedade.
    - [ ] Número + Letra.
    - [ ] Emoji + Texto.

8. O que é uma "Text Property"?
    - [ ] Uma fonte especial.
    - [x] Uma forma de permitir a edição do texto de um componente diretamente no painel de propriedades do lado direito.
    - [ ] O nome do autor.

9. Podemos adicionar interações de animação direto entre as variantes de um componente?
    - [x] Sim, usando o recurso "Interactive Components" no modo de prototipagem.
    - [ ] Não, animações só funcionam entre telas.
    - [ ] Apenas se o arquivo for compartilhado.

10. Se você tem um botão em 3 tamanhos (P, M, G), o mais organizado é:
    - [ ] Criar 3 componentes separados.
    - [x] Criar 1 componente com uma variante chamada "Size" com os valores "Small", "Medium" e "Large".
    - [ ] Não usar componentes para tamanhos.
